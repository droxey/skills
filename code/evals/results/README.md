# Trigger Eval Results Format

Generated reports should be written as:

- `code/evals/results/YYYY-MM-DD-trigger-eval.md`

Each report should include:

1. Metadata (date, expected file, observed file, thresholds, pass/fail status)
2. Confusion matrix (TP, FP, TN, FN)
3. Metrics (precision, recall, accuracy)
